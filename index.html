Looking Back at HTML

Naming Conventions

Files:
File names must only include lowercase letters, numbers, and dashes. Dashes are word separators, while underscores are word connectors.

Good Examples:

Good Examples:

  • chocolate-cake-recipe.html
  • west-coast-art-2021.jpeg
  • snow_white-apple.jpeg

Bad Examples:

Bad Examples:

  • pureKitchenLogo.svg
  • Today-in-History.html
  • picture of black cat.jpg
Paths:
The purpose of a file path is to include a file within your website. Paths open and close on their own.

<img src="image link" alt="image description>"

<img src="image link" alt="image description>"

Adds an image to website. If image doesn't load, image description will act as a placeholder. In addition, the image description is read by text to speech for the visually impaired.

Commit Messages

  • Minimum 3 words and 10 characters
  • Proper spelling and grammar
  • Capital start letter
  • No period
  • Must start with imperative verb (ex. Create, Fix, Add, etc.)
  • Describe as if starting with "This commit will..."

    • Paragraphs

    Defines a paragraph of text. Always starts on a new line of code. Browser automatically adds margins above and below defined text.

    <p>insert text here</p>

    List Item

    Individual items within a list.

    <li>Item name</li>

    Unordered list

    List without a numbered order. Opens and closes on its own line. Contains list items that begin with a dot.

    <ul>

    <li>item name here</li>

    </ul>

    Ordered list

    List in a numbered order. Opens and closes on its own line. Contains list items that begin with a designated number.

    <ol>

    <li>item name here</li>

    </ol>

    Quote

    Adds quotation marks around the specified text.

    <q>Me?</q>

    Blockquote

    For large stand alone quotes. Indents specified text.

    <blockquote>

    Lorem ipsum dolor sit amet, consectetur adipiscing elit, sed do eiusmod tempor incididunt ut labore et dolore magna aliqua. Ut enim ad minim veniam, quis nostrud exercitation ullamco laboris nisi ut aliquip ex ea commodo consequat.
    </blockquote>

Structuring Content

Header

The header is the intro content at the top of the page. It usually includes a logo and main navigation.

<header> </header>

Footer

The bottom content of a webpage. Usually includes links, copyright info, extra info about the company, terms and services.

<footer> </footer>

Main

The main wraps around the main content of the page. It can only be used once per page.

<main> </main>

Section

Wraps around a related piece of content.

<section> </section>

Article

Wraps around a self-contained composition.

<article> </article>

Navigation

Provides navigation in the form of links.

<nav> </nav>

Linking Elements Same Page

Comments

Comments aren't displayed in the browser, however they are seen in the HTML. This is used to leave notes and sources.

<!-- write message here -->

Landmark Roles

Header Role

<header role="banner">

This role defines an element as the masthead. It's added to the header.

Navigation Role

<nav role="navigation">

This role defines an element as the primary navigation. It's added to the top nav.

Main Role

<main role="main">

This role defines an element as the main content of the page. It's added to the main.

Footer Role

<footer role="contentinfo">

This role defines an element as information about the content. It's added to the footer.

Aria Labels

The purpose of aria labels is to add a non-visual label to links. The label will only be read by screen-readers.

<a href="#"aria-label="description of link">Visible text</a>
